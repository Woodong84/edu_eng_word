# 영단어 마스터

초등학생용 영단어 학습 웹앱. 빌드 도구 없이 브라우저에서 바로 실행되는 정적 페이지이며, 모바일 사용을 우선으로 설계되어 있습니다.

## 주요 기능

- **프로필(로그인) 관리**: 비밀번호 없이 이름+아이콘으로 프로필을 만들어 전환. 상위/하위 계정(부모-자녀) 구조 지원. 프로필별로 단어장·레벨·진도가 분리 저장됨
- **레벨/보상 시스템**: 정답 시 EXP 획득, 레벨업 시 풀스크린 이펙트와 스티커 지급. 레벨 구간(새싹→탐험가→마법사→챔피언→마스터)에 따라 앱 색상과 배경이 자동으로 변경. 출석 스트릭·연속 정답 콤보·업적 배지 32종 지원
- **단어 폴더 관리**: 프로필별로 폴더(학습장)를 만들어 단어를 분류. 목록 필터링, 폴더 단위 학습 범위 지정 지원
- **받아쓰기 학습**: TTS로 단어를 읽어주고 스펠링 입력(뜻 함께 표시). 틀릴수록 힌트 공개, 오답 노트 자동 누적
- **초등 필수 단어 300개 프리셋**: 카테고리별 선택 추가
- **마스터 단어장 연동 (읽기 전용)**: GitHub raw 파일에서 공용 단어 목록을 읽어와 병합. 등록할 때마다 중복/형식 검증을 거쳐 데이터 정합성 유지
- **클라우드 연동 (Firebase)**: 프로필/통계는 Firestore로 동기화되어 **주간/월간 랭킹**을 볼 수 있고, 문제 풀이마다 **사용 이력**이 기록됩니다. 관리자 계정으로 로그인하면 **Word 마스터 데이터**를 Firestore에 등록할 수 있습니다. 상세 구조는 [`docs/SPEC.md`](docs/SPEC.md) §5 참고

## 실행 방법

저장소를 내려받은 뒤 `index.html`을 브라우저로 열거나, 정적 서버로 서빙하면 됩니다.

```bash
python3 -m http.server 8000
# http://localhost:8000/index.html 접속
```

## 디렉토리 구조

```
.
├── index.html            # 마크업 (프로필 화면 + 학습 앱 + 오버레이)
├── firestore.rules       # Firestore 보안 규칙 소스 (콘솔에 수동 게시 필요)
├── css/
│   └── style.css         # 모바일 우선 스타일, 레벨 구간별 테마
└── js/
    ├── storage.js        # 저장소 추상화 계층 (Adapter 패턴)
    ├── profiles.js       # 로컬 프로필(계정) 관리, 상위/하위 계정, 마이그레이션
    ├── firebase-init.js  # Firebase SDK 초기화 (CDN ESM 모듈)
    ├── cloud.js          # 로그인 관리/마스터 단어 DB/사용이력/랭킹 (Firestore)
    ├── app.js            # 화면 로직: 단어장/폴더/퀴즈/보상/마스터 연동
    └── data/
        └── elementary-words.js  # 초등 필수 단어 300개 프리셋 (카테고리별)
```

문서: [`docs/SPEC.md`](docs/SPEC.md)(사양서), [`docs/PROMPT_HISTORY.md`](docs/PROMPT_HISTORY.md)(요청/변경 이력)

## 데이터 구조 (localStorage)

모든 키는 `eduword:v2:` 프리픽스로 저장됩니다.

| 키 | 내용 |
|---|---|
| `profiles` | 프로필 목록 `[{id, name, icon, parentId, createdAt}]` |
| `activeProfileId` | 현재 선택된 프로필 ID |
| `cloudConfig` | 마스터 단어장 연동 설정 (rawUrl, token) — 프로필 공통 |
| `p:<id>:words` | 프로필별 단어장 `[{id, word, addedDate, correctCount, incorrectCount, folderId, source}]` |
| `p:<id>:stats` | 프로필별 레벨/EXP/스티커 |
| `p:<id>:folders` | 프로필별 단어 폴더 목록 |

기존(v1) 데이터가 있으면 첫 실행 시 기본 프로필로 자동 마이그레이션됩니다.

## Firebase 연동

프로필/통계·마스터 단어·사용이력·랭킹은 Firebase(Firestore)로 동기화됩니다. 개인 단어장/폴더는 여전히 기기의 `localStorage`에만 저장됩니다(오프라인에서도 학습 기능은 그대로 동작). `firestore.rules`는 저장소에 소스로만 보관되며, 변경 시 Firebase 콘솔 → Firestore → 규칙 탭에 수동으로 붙여넣고 게시해야 합니다. 인증 모델, 데이터 흐름, 실패 내성 설계는 [`docs/SPEC.md`](docs/SPEC.md) §5에 상세히 정리되어 있습니다.

## 알려진 제약

- 개인 단어장/폴더는 기기(브라우저) 안에서만 유효하며 기기 간 동기화되지 않습니다 (프로필/통계/마스터 데이터/이력/랭킹만 클라우드로 동기화됨)
- 로그인에 비밀번호가 없으므로 보안 목적이 아닌 "학습자 구분" 용도입니다
- Firestore 보안 규칙은 가족 전체를 하나의 신뢰 그룹으로 취급합니다 (불특정 다수 대상 서비스에는 부적합)
