# 영단어 마스터

초등학생용 영단어 학습 웹앱. 빌드 도구 없이 브라우저에서 바로 실행되는 정적 페이지이며, 모바일 사용을 우선으로 설계되어 있습니다.

## 주요 기능

- **프로필(로그인) 관리**: 비밀번호 없이 이름+아이콘으로 프로필을 만들어 전환. 상위/하위 계정(부모-자녀) 구조 지원. 프로필별로 단어장·레벨·진도가 분리 저장됨
- **레벨/보상 시스템**: 테스트 정답 시 EXP 획득, 레벨업 시 풀스크린 이펙트와 스티커 지급. 레벨 구간(새싹→탐험가→마법사→챔피언→마스터)에 따라 앱 색상과 배경이 자동으로 변경
- **단어 폴더 관리**: 프로필별로 폴더를 만들어 단어를 분류. 학습 목록 필터링, 폴더 단위 테스트 지원
- **받아쓰기 테스트**: TTS로 단어를 읽어주고 스펠링 입력. 틀릴수록 힌트 공개, 오답 노트 자동 누적
- **초등 필수 단어 300개 프리셋**: 카테고리별 선택 추가
- **마스터 단어장 연동 (읽기 전용)**: GitHub raw 파일에서 공용 단어 목록을 읽어와 병합. 등록할 때마다 중복/형식 검증을 거쳐 데이터 정합성 유지

## 실행 방법

저장소를 내려받은 뒤 `index.html`을 브라우저로 열거나, 정적 서버로 서빙하면 됩니다.

```bash
python3 -m http.server 8000
# http://localhost:8000/index.html 접속
```

## 디렉토리 구조

```
.
├── index.html            # 마크업 (프로필 화면 + 학습 앱 + 오버레이)
├── css/
│   └── style.css         # 모바일 우선 스타일, 레벨 구간별 테마
└── js/
    ├── storage.js        # 저장소 추상화 계층 (Adapter 패턴)
    ├── profiles.js       # 로컬 프로필(계정) 관리, 상위/하위 계정, 마이그레이션
    ├── app.js            # 화면 로직: 단어장/폴더/퀴즈/보상/마스터 연동
    └── data/
        └── elementary-words.js  # 초등 필수 단어 300개 프리셋 (카테고리별)
```

## 데이터 구조 (localStorage)

모든 키는 `eduword:v2:` 프리픽스로 저장됩니다.

| 키 | 내용 |
|---|---|
| `profiles` | 프로필 목록 `[{id, name, icon, parentId, createdAt}]` |
| `activeProfileId` | 현재 선택된 프로필 ID |
| `cloudConfig` | 마스터 단어장 연동 설정 (rawUrl, token) — 프로필 공통 |
| `p:<id>:words` | 프로필별 단어장 `[{id, word, addedDate, correctCount, incorrectCount, folderId, source}]` |
| `p:<id>:stats` | 프로필별 레벨/EXP/스티커 |
| `p:<id>:folders` | 프로필별 단어 폴더 목록 |

기존(v1) 데이터가 있으면 첫 실행 시 기본 프로필로 자동 마이그레이션됩니다.

## 향후 클라우드 DB 도입 가이드

앱 로직은 저장소에 직접 접근하지 않고 `js/storage.js`의 `AppStorage` 인터페이스(`get`/`set`/`remove`/`keys`)만 사용합니다. 클라우드 DB(Firebase, Supabase 등)를 붙일 때는:

1. 동일 인터페이스를 구현한 클라우드 어댑터를 작성
2. `AppStorage.setAdapter(cloudAdapter)`로 교체
3. 권장 전략: 로컬 캐시에 즉시 반영 후 백그라운드로 원격 동기화 (오프라인 대응)

단어 데이터에 `source` 필드('manual'/'preset'/'master')가 있어 동기화 시 출처별 처리가 가능하며, 키가 프로필 ID 단위로 분리되어 있어 사용자 테이블로 자연스럽게 매핑됩니다. 진짜 다중 사용자 인증(비밀번호)과 "마스터 계정이 다른 기기의 하위 계정을 관리"하는 기능은 클라우드 DB 도입 시점에 구현 가능해집니다.

## 알려진 제약 (서버리스 구조)

- 프로필은 기기(브라우저) 안에서만 유효하며 기기 간 동기화되지 않습니다
- 로그인에 비밀번호가 없으므로 보안 목적이 아닌 "학습자 구분" 용도입니다
