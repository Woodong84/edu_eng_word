# 프롬프트/작업 이력 (edu_eng_word)

이 문서는 `woodong84/edu_eng_word` 저장소의 "영단어 마스터" 앱을 만들면서 오간 요청과, 각 요청에 대해 실제로 적용한 변경 사항을 시간 순서로 정리한 기록입니다. 새로 이 프로젝트를 이어받는 사람이 "왜 이런 구조인지"를 빠르게 파악할 수 있도록 하는 것이 목적입니다.

각 항목은 `사용자 요청(요약)` → `적용 내용` → `관련 PR` 형식으로 정리했습니다. 실제 커밋 메시지와 PR 본문에 더 상세한 내용이 있습니다.

## 5. 추가 기능 타당성 검토 (DB 없는 구조 기준)

**요청**: 로그인 관리(사용자별/하위 계정/마스터 계정), 사용자별 계정·레벨 관리(아이콘, 레벨별 색/배경, 레벨업 이펙트), 단어 추가 관리(사용자별 등록, 폴더 관리), 마스터 데이터 등록 시 중복 검증 — 이 네 가지를 **별도 DB 없이** 적용 가능한지 항목별로 검토하고, 가능하면 시작 전에 확인을 받아달라.

**적용**: 코드 변경 없이 항목별 가능/불가 표로 검토 결과 제시.
- **마스터 계정이 다른 사용자 기기의 데이터를 열람/통제하는 기능은 DB(서버) 없이는 불가능**하다고 명확히 함 (각자 데이터가 각자 브라우저에만 존재하기 때문)
- 나머지(로그인=로컬 프로필 전환, 계정/레벨/아이콘, 단어 폴더, 중복 검증)는 localStorage 구조로 구현 가능하다고 판단
- `AskUserQuestion`으로 마스터 계정 범위와 로그인 방식(비밀번호 없는 프로필 전환)에 대해 확인 요청 → 사용자가 이후 턴에서 "불가능 항목 제외, 나머지 전부 적용" + "모바일 우선 고려" + "리팩토링" + "향후 클라우드 DB 도입 고려"로 답함

## 7. 모바일 TTS(소리) 무음 버그 수정 (PR #2)

**요청**: PC에서는 스피커 아이콘으로 다시 듣기가 되는데 모바일에서는 소리가 안 남. 원인 확인 후 수정, 스피커 클릭 후 정답 입력창으로 자동 포커스 이동 추가.

**적용**:
- 원인: `speechSynthesis.cancel()` 직후 곧바로 `speak()`를 호출하면 모바일 크롬이 새 재생 요청을 조용히 무시하는 버그가 있음
- 재생 중일 때만 cancel 후 80ms 지연 뒤 재생, 유휴 상태에서는 사용자 제스처 컨텍스트 안에서 즉시 재생(iOS 정책 대응), paused 상태면 resume 처리, utterance 참조 유지(크롬 GC 중단 버그 방지)
- 스피커 버튼 클릭 시 정답 입력창(`#quiz-input`)으로 자동 포커스

**관련 PR**: #2 Fix silent TTS replay on mobile and auto-focus answer input

## 부록: 캐시 무효화

3번 기능 개선(색상 구분)이 배포 후에도 모바일에서 반영되지 않는 문제가 있었는데, 원인은 모바일 브라우저가 `css/style.css`, `js/app.js` 등을 강하게 캐시하고 있었기 때문이었습니다. 이후 모든 정적 자원 참조에 `?v=N` 쿼리 파라미터를 붙이고, 배포할 때마다 숫자를 올리는 방식(현재 `v=14`)으로 강제 갱신되도록 조치했습니다. 새 버전을 배포한 뒤에도 화면이 그대로면 우선 브라우저 새로고침(캐시 삭제)을 안내하는 것이 좋습니다.
